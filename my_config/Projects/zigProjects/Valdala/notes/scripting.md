# Scripting

## Language candidates


## Lua

https://www.lua.org/

http://luajit.org/

https://luau.org/


advantages:
- well known
- fast (proven, at least with LuaJit)
- simple

disadvantages:
- anemic type system
- quirky syntax with gotchas
- lots of variants with different features and performance characteristics


## Umka

https://github.com/vtereshkov/umka-lang

advantages
- static types
- interfaces
- fast (it says so)
- relatively well readable

disadvantages
- some weird syntax
- not well known
- questionable support


## Cyber

https://cyberscript.dev/

advantages
- static types
- fast (it says so)
- well readable

disadvantages
- not well known
- lately very infrequent updates


## Berry

https://berry-lang.github.io/

advantages
- fast (it says so)
- well readable

disadvantages
- not well known
- questionable support
